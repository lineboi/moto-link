# Moto-Link · Lyftathon Kigali 2026

**Voice-first vernacular navigation for Kigali's moto-taxi drivers.**

Moto-Link solves a problem that every moto-taxi driver in Kigali faces: formal addresses don't exist for most destinations. Passengers give directions using local landmarks, slang, and neighbourhood nicknames — _"kuri pavé hafi ya Kwa Nyiranuma"_ — that no standard GPS app understands. Moto-Link captures these vernacular descriptions via voice, transcribes them with the **Hugging Face `mbazaNLP/Whisper-Small-Kinyarwanda`** speech-to-text model, and passes the transcript to the **Claude API** (configured as a Kigali Geographic Expert) to extract up to 5 ranked destination candidates with GPS coordinates. Results are cross-referenced against a live Supabase `vernacular_landmarks` database so driver-confirmed landmarks receive real coordinates, not just AI estimates. The driver taps a result card, and a full-screen **MapLibre GL** map appears with their real-time GPS position, an amber route polyline fetched from the public **OSRM** routing engine, and glove-friendly "Start / Pause / End Trip" session controls.

## Offline PWA — The Demo Will Not Crash

The app is a fully configured **Progressive Web App** powered by `vite-plugin-pwa` and a Workbox service worker. Every JavaScript bundle, CSS file, and the CARTO Dark Matter vector-tile style JSON are **precached at install time**. On subsequent loads — even with no network connection — the React app shell, the Chakra UI design system, and the MapLibre map engine all load instantly from the browser cache. Drivers' most recent search results are persisted to **IndexedDB** via `localforage` (Zustand's `persist` middleware with a `partialize` filter that only serializes `landmarkResults` and `language`). If a driver enters a 4G dead zone after a search, the amber _"Offline — Using cached results"_ banner appears and they can still tap a cached destination card, view the map with their live GPS position, and navigate to recently searched places — the app never crashes or shows a blank screen. On Android, the PWA installs to the home screen with a single tap, runs in `display: standalone` mode, and respects the moto-themed `#06122A` dark navy status bar.

## Neural Refinement — The AI Gets Smarter Every Trip

The most technically significant feature is the **self-healing landmark database**. Every trip inserts a row into the Supabase `navigation_sessions` table with one of three statuses: `CONFIRMED` (driver arrived at the correct destination), `REJECTED` (result was immediately wrong), or `ABANDONED` (trip was cancelled mid-route). A **PostgreSQL trigger** — `trigger_refine_confidence` on the `navigation_sessions` table — fires after every insert and automatically adjusts the `confidence_score` of the associated `vernacular_landmarks` row: `+0.05` for `CONFIRMED` (capped at 1.0), `−0.10` for `REJECTED`, and `−0.02` for `ABANDONED`. This means every driver interaction makes the system smarter — highly-confirmed vernacular phrases rise to the top of future search results, while misclassified ones fade out. The `reinforcement_dataset` table captures every raw transcript alongside the confirmed GPS coordinates, creating a training dataset for future fine-tuning of the Whisper and Claude models. To see the Neural Refinement in action during the demo: open the **Supabase Table Editor → vernacular_landmarks** on a second screen and watch `confidence_score` and `successful_routes` update in real-time as trips are confirmed.
